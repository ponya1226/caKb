# ADR 0004: 店舗別カテゴリルール

Date: 2026-07-05

## Status

Accepted

## Context

Google Vision OCRにより文字認識精度は改善したが、家計簿入力ではOCR後のカテゴリ選択を毎回行う負担が残る。買い物する店舗はある程度固定されるため、一度登録した店舗のカテゴリを次回以降の初期値に使えると、確認画面の修正量を減らせる。

既存実装では、保存済み支出の店舗名から直近カテゴリを推定している。ただし、ユーザーが明示的に管理できるルールではないため、意図しないカテゴリが反映された場合に確認、編集、削除しづらい。

## Decision

店舗別カテゴリルールを `AppSettings.shopCategoryRules` としてlocalStorageに保存する。IndexedDB schemaは変更しない。

ルールは店舗名、正規化店舗名、カテゴリID、作成日時、更新日時を持つ。カテゴリ初期値の判定では、店舗別カテゴリルールを保存済み支出履歴より優先する。

OCR確認画面では、保存時に「この店舗のカテゴリを次回も使う」を選べるようにする。設定画面では、店舗別カテゴリルールの追加、カテゴリ変更、削除をできるようにする。

## Consequences

- 支出データ、レシート画像、OCR全文のIndexedDB schemaは変更しない。
- JSONバックアップにはsettings経由で店舗別カテゴリルールが含まれる。
- CSVエクスポートには店舗別カテゴリルールを含めない。
- ルールは端末内保存であり、ログイン、クラウド同期、家族共有は追加しない。
- 店舗名の一致は既存の正規化と部分一致ヒューリスティックを使うため、完全な店舗識別ではない。

## Alternatives

- 保存済み支出履歴のみを使う: 実装は単純だが、ユーザーがルールを管理できない。
- IndexedDBに専用storeを追加する: 将来の拡張性はあるが、現時点ではschema migrationの負担が大きい。
- クラウド側で店舗辞書を管理する: MVPのローカル保存方針から外れるため採用しない。
